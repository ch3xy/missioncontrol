# Development Plan

## Phase 1 — Foundation

Goal:
Create a runnable Angular app with shell and routing.

Deliverables:
- App shell
- Sidebar
- Header
- Routes
- Dark theme tokens

## Phase 2 — Dashboard MVP

Goal:
Implement the Visual 3-inspired dashboard.

Deliverables:
- Widget grid
- Calendar card
- Velo card
- Dash cards
- Quick access cards

## Phase 3 — Data Layer

Goal:
Use typed mock data through adapter services.

Deliverables:
- Interfaces
- Mock adapters
- Service methods
- Loading/error states

## Phase 4 — Settings

Goal:
Make external app URLs configurable.

Deliverables:
- Settings page
- LocalStorage persistence
- Defaults
- Reset function

## Phase 5 — Detail Views

Goal:
Add useful placeholder pages.

Deliverables:
- Calendar overview
- Velo overview
- Dash overview

## Phase 6 — Polish

Goal:
Make MVP feel cohesive and robust.

Deliverables:
- Responsive layout
- Empty/error states
- Build fixes
- Lightweight tests
- Documentation updates

## Definition of Done

The MVP is done when:
- It runs locally
- It builds without TypeScript errors
- Dashboard is visually close to Visual 3
- Settings persist
- External shortcuts work
- No backend is required
