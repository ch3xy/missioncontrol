# Mission Control — Codex Implementation Pack

Dieses Paket ist für eine möglichst selbstständige Umsetzung mit Codex optimiert.

Ziel: Eine lokale, frontend-only Angular-22-Applikation namens **Mission Control** als zentrales Dashboard für einen selbstständigen Softwareentwickler.

## Wichtigste Dateien

- `AGENTS.md` — zentrale Arbeitsanweisung für Codex
- `CODEX_MASTER_PROMPT.md` — Prompt für eine weitgehend autonome Umsetzung
- `CODEX_TASKS.md` — konkrete Entwicklungsaufgaben mit Akzeptanzkriterien
- `ARCHITECTURE.md` — technische Architektur
- `UI_UX_SPEC.md` — Designvorgabe basierend auf Visual 3
- `DATA_CONTRACTS.md` — TypeScript-Datenmodelle und Adapter-Verträge
- `DEVELOPMENT_PLAN.md` — Entwicklungsplan in Phasen
- `NO_BACKEND_DECISION.md` — Frontend-only-Entscheidung und Grenzen
- `PROJECT_STRUCTURE.md` — empfohlene Ordnerstruktur
- `IMPLEMENTATION_CHECKLIST.md` — Abschluss-Checkliste
